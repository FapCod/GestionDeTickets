## üóÑÔ∏è Database Schema

 The system is built on a relational PostgreSQL schema (Supabase):

-   **`modules`**: Core functional areas of the application.
-   **`releases`**: Linked to Modules, defining release cycles.
-   **`tickets`**: Work items linked to statuses, teams, developers, and environments.
-   **`components`**: Technical pieces belonging to a Module.
-   **`ticket_component_matrix`**: The intersection table defining which components are affected by which ticket.
-   **`developers`**: Pivot table associating developers with specific Modules.

## üöÄ Getting Started

1.  **Clone the repository**
    ```bash
    git clone [repository-url]
    cd [project-name]
    ```

2.  **Install dependencies**
    ```bash
    npm install
    ```

3.  **Environment Setup**
    Create a `.env.local` file in the root directory:
    ```env
    NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
    ```

4.  **Database Setup**
    Run the SQL scripts located in `.gemini/` in your Supabase SQL Editor in the following order:
    1.  `01_init_schema.sql` (Tables & Relations)
    2.  `02_seed_data.sql` (Initial Data)
    3.  `05_developers_modules.sql` (Developer scoping)
    4.  `06_add_responsible_to_releases.sql` (Release enhancements)

5.  **Run Development Server**
    ```bash
    npm run dev
    ```

    Open [http://localhost:3000](http://localhost:3000) to view the application.
